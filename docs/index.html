SCBL

为社区联机环境提供 Windows 客户端、Linux 服务端一键部署、EasyTier 私有网络和完整客户端更新发布。每位管理员拥有完全独立的自建网络。

下载 Windows 客户端 查看源代码 本地编译

快速开始

Windows 用户从 Release 下载预编译完整包；开发者可在本地调用同一套构建脚本；Linux 服务端通过一条命令部署。

Windows 客户端

前往 Release 下载 SCBL-Client-v0.6.0-win-x86.zip，解压后运行启动器。仓库不提供 Windows 在线安装脚本。

打开 Releases →

Windows 本地编译

需要 Windows 10/11、.NET 8 SDK、Go 1.23.x、Git 和 PowerShell。

git clone https://github.com/caox233/SCBL.git
cd SCBL\client
powershell -ExecutionPolicy Bypass -File .\build_all_windows.ps1
powershell -ExecutionPolicy Bypass -File .\create_client_full_package.ps1

Linux 服务端

适用于 Linux x86_64。安装器会校验服务端部署包，并按 SHA256 判断是否复用现有专版游戏服务端。

curl -fsSL https://github.com/caox233/SCBL/releases/latest/download/install-server.sh | sudo bash

独立自建架构

没有统一官方网络。每个部署者拥有自己的服务器、网络密码、数据库、控制平面和客户端更新源。

Windows 客户端→公网更新 TCP/18080→EasyTier TCP/UDP 11010→10.66.0.0/24 私网→游戏服务与控制平面

客户端

WPF 启动器负责网络编排、账号、公告和更新；EasyTier 使用官方未修改二进制；Route Guard 仅约束游戏虚拟网段流量。

服务端

部署脚本安装 systemd 服务、EasyTier、控制平面、更新文件服务和专版 dedicated_server。安装后生成 /opt/scbl-public/client_launcher_settings.json，管理员安全分发给自己的客户端用户。

先检查更新，再接入私网

v0.6.0 不改变现有 manifest、公告、差异下载、Updater、回滚和完整包发布方式，只调整执行顺序。

启动→公网主机:18080 检查→有更新：原流程更新并重启→无更新：启动 EasyTier
公网检查失败不会阻止客户端启动。EasyTier 私网连通后，客户端会继续使用原 10.66.0.1:18080 地址兜底检查。TCP/11010 已由 EasyTier 占用，不能同时承载现有 HTTP 更新服务。

服务端发布完整客户端

GitHub Actions 编译完整客户端 ZIP
→ 管理员从 Release 下载或在服务器同步
→ 放入 /opt/scbl-public/incoming/client
→ scbl-package-watch.timer 每 60 秒自动发布
→ 客户端读取原 client_update_manifest.json

端口规划

端口协议用途范围

10443TCPEasyTier WSS 备用接入公网
18080TCP/HTTP现有客户端 manifest 与更新文件公网读取
19080TCP/HTTPSCBL 控制平面私网
50051TCP/gRPC账号服务私网

GitHub 自动发布

源码和正式二进制分离。普通用户只需 Release；开发者可自行编译并对照 CI 结果。

验证

提交时检查版本一致性、Shell、Python、控制平面测试、Go vet/test 和 Windows Route Guard 交叉编译。

客户端构建

Windows Server 2022 Runner 自动安装固定 .NET 和 Go 版本，调用 client/build_all_windows.ps1 生成完整 ZIP。

正式 Release

推送语义版本标签后生成客户端 ZIP、Linux 服务端包、SHA256、manifest 和一键安装脚本。已有版本不允许覆盖。

SCBL 是非官方社区项目。正式公开前必须确认 dedicated_server、WinDivert 和客户端嵌入文件的许可证及再分发权限；不得包含游戏本体或未经授权的版权资源。
